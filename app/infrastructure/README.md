# app/infrastructure/

**Инфраструктурный слой (Infrastructure Layer).** Содержит конкретные реализации всего, что связано с внешним миром: база данных, сторонние API, очереди, кэш. Этот слой реализует интерфейсы, определённые в `app/domain/`.

## Структура

```
infrastructure/
├── database/
│   ├── models/         # SQLAlchemy ORM-модели (таблицы)
│   └── repositories/   # Конкретные реализации AbstractRepository
└── external/           # Клиенты внешних HTTP-сервисов
```

## Правило изоляции

Инфраструктурный слой **знает** про домен (импортирует сущности и интерфейсы), но домен **не знает** про инфраструктуру. Это позволяет менять БД или внешние сервисы, не трогая бизнес-логику.

## Что сюда попадает

- ORM-модели (SQLAlchemy `DeclarativeBase`) — отражают структуру таблиц БД
- Реализации `AbstractRepository` с реальными SQL-запросами через SQLAlchemy async
- Клиенты внешних сервисов (платёжные системы, CRM, ERP, email/SMS-рассылки)
- Адаптеры/маперы: конвертация ORM-модель ↔ доменная сущность

## Чего сюда НЕ попадает

- Бизнес-логика и правила домена
- HTTP-роутеры и Pydantic-схемы запросов
- Конфигурация приложения (она в `app/core/`)
